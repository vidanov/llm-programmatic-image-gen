# Contributing

This repo is a collection of working examples proving that LLMs writing rendering code is 100x cheaper than diffusion models. Contributions that expand the renderer spectrum, add benchmarks, or improve existing examples are welcome.

## What would help most

1. **Blender Python example**: a product visualization or architectural render generated by LLM-written `bpy` code
2. **Godot .tscn example**: a game scene or UI layout as a text-based scene file
3. **Manim video example**: a math explainer that renders to MP4
4. **Model benchmarks**: test Gemini Flash, GPT-4o, or other models with the same crayon prompt
5. **Cost data**: real token counts and timing from production use

## Adding an example

1. Create a self-contained file in `examples/` (HTML, SVG, or Python script)
2. Include a comment header with: what it demonstrates, approximate token count, estimated cost
3. If it requires a runtime (Blender, Godot, Manim), include the exact command to render
4. Add an entry to the Examples table in README.md

## Style

- HTML/JS examples must be self-contained (no build step, no npm)
- SVGs must use system font stacks only (no Google Fonts, no external resources)
- Python scripts should run with standard library + one renderer (bpy, manim, etc.)
- Keep token estimates honest: count the actual output, not a guess

## Articles

Article drafts live in `articles/`. If you want to write one:
- Follow the existing series format (frontmatter, cover image, series tag)
- Include at least one working code example
- Include a cost comparison table
- Link to the live demo and repo

## Process

1. Fork and branch
2. Make your changes
3. Test that examples render correctly
4. Submit a PR with a clear description of what the example demonstrates

No issue required for small additions. For larger proposals (new renderer category, architectural changes), open an issue first.
