Image generation APIs charge $0.02-0.20 per image. Meanwhile, I've been using Claude's artifacts to create SVG illustrations for months. They're editable, scalable, and the text is always perfect.

I got curious: what does this actually cost on Bedrock? And which use cases can it cover compared to diffusion models?

Turns out: $0.0003 per image. And the list of covered use cases is longer than I expected.

## The pipeline

Three steps. Ten seconds. No diffusion model involved.

```
PROMPT (10 words)
  → "cloud architecture diagram, 6 AWS services"

LLM GENERATES CODE (~800 tokens, 2s)
  → <svg viewBox="0 0 800 600">
      <rect .../>
      <text>API Gateway</text>
      ...
    </svg>

BROWSER RENDERS (0ms)
  → Editable vector image at any resolution
```

The LLM pays for tokens. The renderer is free.

## What it actually costs

I tested this on AWS Bedrock with Claude Sonnet (the cheapest model that produces good SVGs):

```python
import boto3, json

bedrock = boto3.client('bedrock-runtime', region_name='us-east-1')

response = bedrock.invoke_model(
    modelId='us.anthropic.claude-sonnet-5-v1',
    body=json.dumps({
        "anthropic_version": "bedrock-2023-05-31",
        "max_tokens": 4096,
        "messages": [{
            "role": "user",
            "content": "Generate an SVG illustration of a cloud architecture with API Gateway, Lambda, DynamoDB, and SQS. Use official AWS colors. Include connection arrows and labels."
        }]
    })
)

svg_code = json.loads(response['body'].read())['content'][0]['text']
# Save to file, open in browser. Done.
```

**Input**: 45 tokens ($0.000135 at $3/1M)
**Output**: ~850 tokens ($0.0000425 at $15/1M for Sonnet, much less for Haiku)
**Total**: ~$0.0003

Compare that to image generation models:

| Service | Cost/image | Text accuracy | Editable |
|---------|-----------|---------------|----------|
| GPT Image 1.5 | $0.034 | Good (2026) | No |
| Nano Banana 2 | $0.067 | Good | No |
| gpt-image-1-mini | $0.011 | Good | No |
| **Claude → SVG** | **$0.0003** | **Perfect** | **Full source** |

At 10,000 images/month: $670 (Nano Banana 2) vs $6 (Claude SVG). Same Bedrock account, same API call pattern, 100x less spend.

## What you get vs. what you lose

**You get:**
- 100% text accuracy (code specifies text, never hallucinates it)
- Full editability (it's source code: change a color, move a node, resize)
- Version control (diff SVGs in PRs, track changes)
- Deterministic output (same prompt, same code, same image)
- Infinite resolution (vectors scale without quality loss)
- $0.0003 per image instead of $0.02-0.20

**You lose:**
- Photorealistic imagery (stick to diagrams, illustrations, data viz)
- Pixel-level artistic texture (no brushstrokes, no film grain)
- One-click "surprise me" creative exploration

For my use case (technical content: architecture diagrams, flow charts, editorial illustrations, data visualizations), what I gain outweighs what I lose by a wide margin.

## When to reach for this

The decision is simple:

- **Architecture diagrams**: always programmatic. Opens in draw.io, every icon correct.
- **Charts, KPIs, dashboards**: always. Parameterize once, render infinite variants.
- **Blog illustrations**: usually. Unless you need photo-realism.
- **Social cards, hero images**: yes. Unsplash photo + LLM text overlay = $0.003.
- **Photorealistic product shots**: no. Use diffusion or photography.
